# Feature 04.5: App Shell, Navbar, and My Polls List

## Goal

Give authenticated users a persistent navigation shell and a home dashboard that lists their polls, making the app navigable without relying on remembering poll IDs.

## Status

Planned.

## Scope

In scope:

- Persistent top navbar for all authenticated pages (logo, nav links, user info, logout).
- `AppShell` layout component wrapping all protected routes.
- Home page replaced with a "My Polls" dashboard listing the user's polls.
- Poll list shows title, status badge, and links to the detail page.
- Browser tab title updated to "SnapVote".

Out of scope:

- Global poll search or discovery (find polls by other users).
- Pagination of the poll list (deferred to a later feature).
- Notification badges or real-time poll count updates.

## User Journeys

1. User logs in and immediately sees a list of their polls with status badges.
2. User clicks a poll to navigate to its detail page.
3. User clicks "New Poll" in the navbar from any page and is taken to the create form.
4. User clicks the logo to return to the home dashboard from any page.
5. User logs out from the navbar without needing to return to the home page first.

## Acceptance Criteria

- Navbar is visible on all authenticated pages (`/`, `/polls/new`, `/polls/:id`).
- Navbar is NOT visible on auth pages (`/login`, `/register`, `/forgot-password`, `/reset-password`) or the public join page.
- Home page shows all polls owned by the current user (title + status badge + link).
- Empty state is shown when the user has no polls.
- "New Poll" action is accessible from the navbar.
- Logout is accessible from the navbar.

## API Touchpoints

- `GET /polls` — list polls owned by the current user (new endpoint).

## Data Touchpoints

- No schema changes required.
- Query filters by `ownerId` of the authenticated user.

## Frontend Touchpoints

- New `AppShell` component wrapping protected routes in `app.tsx`.
- New `Navbar` component with logo, nav links, and user/logout controls.
- Updated `home.page.tsx` to display the poll list with loading and empty states.
- New `useGetMyPollsQuery` RTK Query hook.
- `GET /polls` endpoint and hook exported from `@libs/client-server-communication`.
- `apps/client/index.html` title updated to "SnapVote".

## Testing

- Unit tests for `AppShell` render and `Navbar` render.
- Updated home page spec to cover poll list, loading state, and empty state.
- Server unit test for the `GET /polls` endpoint (owner filter).

## Technical Debt

Current debt:

- Home page is a placeholder card with no functional content.
- Navigation relies on users knowing poll IDs after navigating away.

Planned debt risk:

- Poll list is unbounded — could grow large for prolific users.

Mitigation:

- Limit initial fetch to latest 50 polls; add pagination in a follow-up feature.
